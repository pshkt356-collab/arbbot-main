
# 🤖 Advanced Crypto Arbitrage Bot

Профессиональный Telegram-бот для арбитража криптовалютных фьючерсов с поддержкой множества пользователей, персональных настроек и автоматической торговли.

## 🌟 Возможности

### 📊 Мониторинг
- **Real-time сканирование** спредов между Binance, Bybit, OKX через WebSocket
- **Персональные настройки** комиссий для каждого пользователя (VIP уровни)
- **Фильтрация пар** по белому/чёрному списку
- **История спредов** для аналитики

### 🔔 Алерты
- **Гибкие настройки** минимального спреда (0.05% - 10%)
- **Уровни алертов**: 💡 0.1% | ⚡ 0.5% | 🚀 1%+
- **Учёт комиссий** при расчёте прибыли
- **Разница фандинга** как дополнительный фактор

### ⚡ Авто-торговля
- **Одновременное открытие** позиций на двух биржах
- **Маркет-ордера** для мгновенного исполнения
- **Автоматическое закрытие** при схождении спреда
- **Мониторинг рисков** в реальном времени

### 🛡️ Риск-менеджмент
- **Лимит позиций** на пользователя
- **Макс. размер позиции** настройка
- **Авто-закрытие** при достижении целевого спреда
- **Отслеживание PnL** и фандинга

## 📱 Использование

### Первый запуск
1. Найдите бота в Telegram и нажмите `/start`
2. Перейдите в **⚙️ Настройки**
3. Настройте **💰 Комиссии** (ваши реальные maker/taker)
4. Установите **🔔 Мин. спред** для алертов
5. (Опционально) Добавьте **⚡ API ключи** для торговли

### Команды

| Команда | Описание |
|---------|----------|
| `/start` | Главное меню |
| `/scan` | Ручное сканирование |
| `/status` | Ваш статус и позиции |
| `/settings` | Настройки бота |
| `/trades` | История сделок |
| `/stop` | Остановить алерты |

### Настройка API для торговли

1. **Binance**:
   - Создайте API ключ на [testnet.binancefuture.com](https://testnet.binancefuture.com) (для теста)
   - Права: `Enable Reading`, `Enable Futures`
   - **БЕЗ права на вывод!**

2. **Bybit**:
   - API создаётся в разделе Account → API
   - Тип: `System-generated`
   - Права: `Read-Write`, `Orders`

3. **OKX**:
   - API Keys → Create API Key
   - Разрешить: `Trade` (не Withdraw!)

## ⚠️ Важные замечания

### Безопасность
- 🔒 **Никогда не добавляйте** права на вывод средств!
- 🧪 **Всегда начинайте с тестовых сетей** (sandbox)
- 🔑 **Храните API ключи локально**, не передавайте третьим лицам
- 📊 **Начинайте с малого** — $100-500 на первые сделки

### Риски арбитража
- **Проскальзывание** — цена может измениться между ордерами
- **Ликвидация** — при высоком плече и резких движениях
- **Задержки** — между биржами спред может исчезнуть
- **Комиссии** — учитывайте все издержки (торговые + фандинг)

### Технические ограничения
- Требуется стабильное интернет-соединение
- Бот должен работать 24/7 для мониторинга
- WebSocket может разрываться — бот автоматически переподключается

## 💰 Типы цен в арбитраже

### Mark Price (Справедливая цена)
- **Используется для**: расчёта ликвидации, unrealized PnL
- **Преимущество**: защита от манипуляций, сглаженная
- **Формула**: средняя спот цена + фандинг
- **Рекомендуется** для арбитража

### Last Price (Последняя цена)
- **Используется для**: исполнения ордеров
- **Преимущество**: реальная цена сделки
- **Недостаток**: может отличаться от спота
- **Риск**: манипуляции в моменте

### Index Price (Индексная цена)
- **Используется для**: ориентира спотовой цены
- **Преимущество**: средняя по многим биржам
- **Недостаток**: может отставать

### Разница Mark/Last
```
Если разница > 0.5% — возможна манипуляция или высокая волатильность
Бот автоматически пропускает такие сигналы
```

## 🛡️ Rate Limiting (Защита от бана)

### WebSocket (рекомендуется)
- **Binance**: безлимитно, публичный поток
- **Bybit**: безлимитно, публичный поток  
- **OKX**: безлимитно, публичный поток
- ✅ **Никаких API ключей не нужно**

### REST API (fallback)
- **Binance**: 1200 запросов/мин на IP
- **Bybit**: 50 запросов/сек на IP
- **OKX**: 20 запросов/сек на IP
- ⚠️ **Используем RateLimiter**

### Наши меры защиты
```python
rate_limiter = RateLimiter(max_requests=5, window_seconds=1)
# Максимум 5 запросов в секунду
```
